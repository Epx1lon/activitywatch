---------------------------------------------------------------------------
                     CONFIGURACIÓN DE CLIENTES
--------------------------------------------------------------------------
1. VERIFICAR PERMISOS
  1.1. Verificar usuario actual, abrir PS y ejecutar el comando: $env:USERNAME
  1.2. Verificar si el usuario tiene permisos de admin: net user $env:USERNAME o tbm ejecutar Get-LocalGroupMember -Group "Administradores"
  1.3. Si tiene permisos de admin, entonces abrir PS con privilegios de admin y ejecutar:  net user Soporte Servicios2022. /add
  1.4. Agregar el usuario Soporte a los administradores: net localgroup Administradores Soporte /add
  1.5. Retirar del grupo de administradores al usuario estándar: net localgroup Administradores NombreDelUsuario /delete
2. CAMBIAR EL HOSTNAME
  2.1. Abrir PS como administrador y ejecutar: Rename-Computer -NewName "TuNuevoNombre"
  2.2. Reiniciar PC: Rename-Computer -NewName "TuNuevoNombre" -Restart
3. INSTALAR ACTIVITYWATCH
  3.1. Descargar el instalador desde: https://activitywatch.net/downloads/
  3.2. Ejecutar el instalador para todos los usuarios
4. MODIFICAR ARCHIVO AW-CLIENT.TOML
  4.1. Abrir el archivo aw-client.toml desde PS: notepad $env:LOCALAPPDATA\activitywatch\activitywatch\aw-client\aw-client.toml
  4.2. Modificar los siguientes campos: [server] hostname = "actividad.montgroup.pe" port = "80" [client] commit_interval = 10
5. CREAR RECOLECTOR
  5.1. Crear el archivo recolector.vbs en la siguiente carpeta: C:\Program Files (x86)\ActivityWatch\:
       Set WshShell = CreateObject("WScript.Shell")
       WshShell.Run """C:\Program Files (x86)\ActivityWatch\aw-watcher-window\aw-watcher-window.exe""", 0, False
       WshShell.Run """C:\Program Files (x86)\ActivityWatch\aw-watcher-afk\aw-watcher-afk.exe""", 0, False
  5.2. Ejecutar win + r y escribir: shell:startup y crear el acceso directo wscript.exe "C:\Program Files (x86)\ActivityWatch\recolector.vbs"
  5.3. Verificar que los watchers están corriendo: Get-Process aw-watcher-*
  5.4. Si fuese necesario, se puede detener los watchers con el comando: Stop-Process -Name "aw-watcher*" -Force -ErrorAction SilentlyContinue
6. RESTRINIGIR LA MODIFICACIÓN DEL ARCHIVO AW-CLIENT.TOML
  6.1. Abrir PS como administrador y ejecutar lo siguiente:
       $ruta = "C:\Users\NombreDelUsuarioEstandar\AppData\Local\activitywatch\activitywatch\aw-client\aw-client.toml"
       $acl = Get-Acl $ruta
       $regla = New-Object System.Security.AccessControl.FileSystemAccessRule("NombreDelUsuarioEstandar", "Write", "Deny")
       $acl.AddAccessRule($regla)
       Set-Acl $ruta $acl
7. OCULTAR LA CARPETA ACTIVITY WATCH
  7.1. Desde PS como administrador, ejecutar: Set-ItemProperty -Path "C:\Program Files (x86)\ActivityWatch" -Name Attributes -Value "Hidden"
8. ELIMINAR HISTORIAL DE DESCARGAS

POR SI LA PC ESTA UNIDA A UN DOMINIO QUE YA NO EXISTE Y SE DESEA CAMBIAR EL HOSTNAME
$NuevoNombre = "arodriguez"
# 1. Cambiar la identidad principal del equipo en el Kernel
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\ComputerName\ComputerName" -Name "ComputerName" -Value $NuevoNombre
# 2. Cambiar la identidad a nivel de red (TCP/IP)
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters" -Name "Hostname" -Value $NuevoNombre
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters" -Name "NV Hostname" -Value $NuevoNombre
Write-Host "Bypass de registro aplicado con éxito. Por favor, reinicia el equipo." -ForegroundColor Green

Set wshShell = CreateObject("WScript.Shell")
' 1. Crear el entorno virtual temporal
Set wshEnv = wshShell.Environment("PROCESS")
' 2. Forzar el DNI o ID del empleado como Hostname falso
wshEnv("AW_HOSTNAME") = "71234567"
' 3. Ejecutar los watchers de forma silenciosa
wshShell.Run """C:\Program Files (x86)\ActivityWatch\aw-watcher-window\aw-watcher-window.exe""", 0, False
